# Fiscal Service Handoff - D1 + D2.1

Fecha: 2026-03-09
Proyecto: `fiscal-service`

## 1) Estado funcional actual

Se implemento base fiscal D1 y motor D2.1 sobre arquitectura actual (`core`, `config`, `infrastructure`, `shared`) sin integrar SUNAT/OSE real.

### D1 implementado
- CRUD y reserva para:
  - `fiscal_series`
  - `fiscal_document`
  - `fiscal_event` (lectura)
  - `company_certificate`
  - `fiscal_provider_config`
- Idempotencia de reserva por `companyId + sourceService + idempotencyKey`.
- Reserva segura de correlativo con lock pesimista en serie activa.
- Estado inicial de documento: `RESERVED`.
- `TICKET` bloqueado como tipo fiscal (regla de dominio).

### D1.5 implementado
- Modulo `fiscal_emitter_config` completo:
  - entidad, DTOs, mapper, puertos, adaptadores, servicio, controlador
  - busqueda/listado/default por scope
  - regla de unico default activo por scope
- Migracion creada:
  - `src/main/resources/db/migration/V2__create_fiscal_emitter_config.sql`

### D2.1 implementado
- Motor interno de procesamiento fiscal (pipeline mock):
  1. carga documento+lineas
  2. valida estado
  3. resuelve emitter
  4. resuelve provider
  5. resuelve certificado
  6. genera XML mock
  7. firma mock
  8. envia mock
  9. actualiza estado/campos artefacto
  10. registra eventos

- Endpoint manual de trigger:
  - `POST /fiscal/documents/{id}/process`

- State machine explicita:
  - `RESERVED -> PENDING_XML`
  - `PENDING_XML -> XML_GENERATED`
  - `XML_GENERATED -> SIGNED`
  - `SIGNED -> QUEUED_FOR_SEND`
  - `QUEUED_FOR_SEND -> SENT`
  - `SENT -> ACCEPTED` (mock)
  - errores -> `ERROR`

- Regla de seguridad de reproceso:
  - Si estado esta en `ACCEPTED`, `REJECTED` o `VOIDED`, se salta procesamiento (`PROCESSING_SKIPPED`).
  - En D2.1 solo procesa documentos en `RESERVED`.

## 4) Seguridad y secretos

- No se exponen llaves privadas ni secretos en respuestas.
- Resolucion de certificado encapsulada.
- Mensajes de error con contenido seguro (sin dump de secret/path).
- Enfoque preferido para certificados: `secret_ref` y `password_secret_ref`.
